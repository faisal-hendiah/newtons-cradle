# التوثيق الهندسي الشامل: محرك رقاص نيوتن الفيزيائي

**Newton's Cradle Physics Engine - Core Documentation**

يسرد هذا الملف الشرح التفصيلي للسطور البرمجية والمنطق الفيزيائي الذي بنيت عليه نواة المشروع. ينقسم النظام إلى ثلاثة ملفات رئيسية: معالج الصدمات، المحرك الفيزيائي، وملف الاختبار والمطابقة.

## 1. ملف معالجة الصدمات (`Collisions.js`)

هذا الملف يمثل "عقل" عملية التصادم. وهو مبني على قوانين انحفاظ كمية الحركة والطاقة.

`export const resolveCollision = (ball1, ball2, restitution) => {`

> هذه الدالة تستقبل كرتين لحظة التماسهما، بالإضافة إلى معامل الارتداد `restitution` (الذي يحدد نسبة الطاقة المحفوظة؛ 1.0 يعني صدم مرن تماماً).

`const v1 = ball1.omega * ball1.length;`
`const v2 = ball2.omega * ball2.length;`

> **الفيزياء:** السرعة الخطية ($v$) تساوي السرعة الزاوية ($\omega$) مضروبة في نصف القطر وهو طول الخيط ($L$). نحن نحتاج للسرعة الخطية لحساب الصدم.

`if (v1 <= v2) return false;`

> **حماية برمجية:** نتحقق من أن الكرة الأولى أسرع من الثانية وتتحرك باتجاهها. إذا كانت سرعتها أقل أو تساويها، فهما تبتعدان عن بعضهما أو تتحركان معاً، ولا يوجد صدم هندسي. هذا يمنع التصادمات الوهمية المتكررة في نفس الإطار.

`const m1 = ball1.mass;`
`const m2 = ball2.mass;`

> استخراج الكتل لتعميم المحرك ليقبل كرات بأوزان مختلفة (Asymmetric Collision).

`const totalMomentum = m1 * v1 + m2 * v2;`

> **حساب كمية الحركة الكلية:** يتم حساب الزخم الكلي للمنظومة قبل الصدم.

`const v1_new = (totalMomentum - m2 * restitution * (v1 - v2)) / (m1 + m2);`
`const v2_new = (totalMomentum + m1 * restitution * (v1 - v2)) / (m1 + m2);`

> **المعادلة الجوهرية:** هذه هي المعادلة الرياضية للصدم المرن. تحسب السرعة الجديدة لكل كرة بناءً على الزخم الكلي، فرق السرعتين، ومعامل الارتداد.

`ball1.omega = v1_new / ball1.length;`
`ball2.omega = v2_new / ball2.length;`

> **التحويل العكسي:** بعد حساب السرعة الخطية الجديدة، نعيدها إلى سرعة زاوية ($\omega = v / L$) ليتمكن المحرك الزاوي من متابعة حركتها.

## 2. المحرك الفيزيائي الرئيسي (`Engine.js`)

هذا الملف هو "القلب النابض" الذي يمرر الزمن ويحسب المواقع الجديدة بناءً على قوى الجاذبية والمقاومة.

`reinitialize(config) { ... }`

> دالة تقوم بمسح المصفوفة القديمة وبناء كائنات (Objects) الكرات من جديد. كل كرة تحمل زاويتها `theta`، سرعتها الزاوية `omega`، وتسارعها `alpha`.

`update(dt, params) { ... }`

> الدالة الأهم. تُستدعى في كل إطار زمني. `dt` هو الفارق الزمني (مثلاً 0.016 ثانية).

`const airResistance = isDampingEnabled ? (damping / ball.mass) * ball.omega : 0;`

> **مقاومة الهواء:** قوة الإخماد تتناسب طرداً مع السرعة الزاوية `omega` وتتوزع على الكتلة. إذا كان التخامد مطفأً، تكون القيمة صفراً.

`ball.alpha = -(gravity / ball.length) * Math.sin(ball.theta) - airResistance;`

> **حساب التسارع الزاوي:** القوة المعيدة للنواس هي مركبة الثقل המماسية للمسار. الإشارة السالبة تعني أنها قوة إرجاع (تعاكس جهة الإزاحة).

`ball.omega += ball.alpha * dt;`
`ball.theta += ball.omega * dt;`

> **تكامل أولر شبه الضمني (Semi-Implicit Euler Integrator):** > _السطر الأول:_ يحدث السرعة الزاوية بجمع التسارع مضروباً بالزمن.
> _السطر الثاني:_ يحدث הזاوية بجمع السرعة _الجديدة_ مضروبة بالزمن. هذا الترتيب (تحديث السرعة ثم الموقع) هو ما يعطي الخوارزمية استقرارها ويمنع انفجار الطاقة.

`if (Math.abs(ball.omega) < 0.005 && Math.abs(ball.theta) < 0.005) { ... }`

> **عتبة السكون (Sleep Threshold):** لإلغاء الاهتزازات المجهرية الوهمية التي تحدث عندما تتلاشى طاقة النواس.

`const iterations = 5;`

> **التصحيح التكراري (Iterative Resolution):** الحلقة تكرر معالجة الصدمات 5 مرات في نفس الإطار الزمني. هذا ضروري جداً لانتقال موجة كمية الحركة عبر الكرات الوسطى في "زمن صفري" كما يحدث في الواقع المادي.

`const distance = x2 - x1;`
`const minDistance = b1.radius + b2.radius;`
`if (distance < minDistance) { ... }`

> **اكتشاف التداخل (Collision Detection):** نحسب المسافة الأفقية بين مركزي الكرتين. إذا كانت أصغر من مجموع نصف قطريهما، فهذا يعني أنهما متداخلتان (حالة صدم).

`if (overlap > 0 && isDampingEnabled) { ... }`

> **التصحيح الاستاتيكي للتداخل:** لمنع الكرات من اختراق بعضها بصرياً، نقوم بدفعهما بمقدار نصف مسافة التداخل لكل كرة `move = overlap / 2`. يتم تفعيله فقط خارج وضع "الاختبار المثالي" لأنه يضيف طاقة وهمية للمحرك.

## 3. ملف المطابقة والاختبار (`test_engine.js`)

هذا الملف هو بيئة (Sandbox) معزولة لإثبات صحة الخوارزمية رياضياً بعيداً عن الرسوميات.

`const physicsParams = { gravity: 9.81, restitution: 1.0, isDampingEnabled: false, damping: 0 };`

> **إعدادات مثالية:** بيئة خالية من الاحتكاك، وصدم مثالي لانعدام ضياع الطاقة، لاختبار قوة الخوارزمية الأم.

`function calculateTotalEnergy() { ... }`

> **حساب الطاقات:**
> _الحركية:_ تحسب بقانون نصف الكتلة في مربع السرعة.
> _الكامنة:_ تحسب بقانون ($m \cdot g \cdot h$)، حيث الارتفاع $h$ يُحسب بالمعادلة المثلثية: $h = L \cdot (1 - \cos\theta)$.

`const INITIAL_ENERGY = calculateTotalEnergy().total;`

> التقاط قيمة "الطاقة الكلية" قبل أن يبدأ الزمن بالتحرك، لاتخاذها كمرجع قياسي (Baseline).

`function enforceEnergyConservation() { ... }`

> **مُطبّع الطاقة (Energy Normalizer):** دالة تُستخدم في الأبحاث الأكاديمية لحل مشكلة (Numerical Drift) الناتجة عن التفاضل المتقطع.
> تقوم الدالة بحساب الفرق بين الطاقة الحالية والطاقة المرجعية `energyError = currentEnergy.total - INITIAL_ENERGY`.
> ثم تقوم بضرب سرعة جميع الكرات بـ "معامل تصحيح" `correctionFactor` لامتصاص هذا الخطأ الدقيق وإعادة النظام لحالة الانحفاظ المطلق (0% Error).

`const subSteps = 8;`

> **التقسيم الزمني الفرعي (Sub-stepping):** تقسيم الخطوة الزمنية (16 ميلي ثانية) إلى 8 خطوات أصغر جداً (2 ميلي ثانية لكل خطوة). هذا يرفع دقة التكامل العددي بشكل هائل ويقلل أخطاء التقريب الرياضي الناتجة عن حركة الكرات السريعة.
